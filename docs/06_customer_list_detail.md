指示書 §13, §14 に対応。

## 顧客一覧 `/customers`

### 検索・絞り込み
| フィルタ | 入力UI |
|---|---|
| お客様名 / フリガナ | テキスト（部分一致 + pg_trgm） |
| 電話番号（下4桁） | テキスト → サーバ側で `phone_hash_last4` 比較 |
| 住所 | テキスト（市区町村部分一致） |
| 担当者 | ユーザマルチセレクト |
| 催事会場 | events マルチセレクト |
| 催事日（範囲） | DateRangePicker |
| Looop申込ステータス | セレクト |
| 太陽光連携ステータス | セレクト |
| クロスセル商材 | チェックボックス（光/水/モバイル/太陽光/蓄電池） |
| 同意取得 | 個人情報 / 太陽光提供 のチェック |
| ピン確認 | 確認済 / 未確認 |
| キャンセル | 含む / 除く |

URLクエリで状態保持（共有可能）。

- count は別クエリで（推定で十分なら `EXPLAIN` 経由でも）

## 顧客詳細 `/customers/[id]`

### レイアウト
左: 基本情報カード / 右: タブ
```
[基本情報] [住所・地図] [Looop] [クロスセル] [太陽光連携] [同意履歴] [対応履歴] [ファイル] [監査]
```

### 各タブ

#### 基本情報
- 名前/フリガナ/電話/予備電話/メール/年齢層/世帯構成/連絡希望時間/備考
- 電話番号は ★ボタン押下で表示（押下時に audit log: `view_pii`）
- 編集は権限保有者のみ（インラインEdit）

#### 住所・地図
- Google Map embed (`google_maps_url`)
- 緯度経度 / Place ID / accuracy_status
- 住居種別 / 所有区分
- 「ピン確認画面を再度開く」ボタン（doc 05 の MapPinConfirmer 再利用）

#### Looop
- 申込状況・現在の電力会社・月間電気料金・売上単価・売上計上月・入金ステータス
- ステータス変更でモーダル → 理由必須（特にキャンセル）

#### クロスセル
- 商材ごとの行（光/水/モバイル/太陽光/蓄電池）
- 興味度 A/B/C、ステータス、次回アクション、メモ

#### 太陽光連携
- 第三者提供同意の有無を大きく表示
- 同意あり → 連携ステータス／連携先／連携日時／連携担当者
- 同意なし → 「連携できません」表示。同意取得ボタン

#### 同意履歴
- 個人情報利用 / 太陽光提供 を時系列で表示
- 取得・撤回イベントとバージョン
- 「同意を撤回する」ボタン（権限要、撤回理由必須）

#### 対応履歴
- activities の時系列 + 入力フォーム（電話/訪問/メール/メモ）
- 次回アクション日設定可

#### ファイル
- 電気料金明細・その他添付
- private blob URL → Server Action 経由でセッショントークン付き短期URLを発行（直接署名URLでも可、要審査ログ）

#### 監査
- adminのみ閲覧
- この顧客に対する全監査ログを表示

## 編集の競合制御
- 楽観ロック: `updated_at` を hidden で送り、サーバで一致確認
- 不一致なら「他のスタッフが更新しました」モーダル
